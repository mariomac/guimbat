# Guimbat — Agent Context

This file provides context for AI coding assistants working on this codebase.

## What this is

A macOS terminal-style calculator app. The user types Numbat expressions into an
input field; results appear in a scrollable, syntax-highlighted, read-only history
pane above. Keyboard-driven: arrow-up/down recalls history, Enter submits,
`clear` and `quit`/`exit` are built-in commands.

## Stack

| Layer        | Crate / tool               | Notes                                      |
|-------------|----------------------------|--------------------------------------------|
| UI framework | `iced = "0.13"`            | Elm-style: state + Message + update + view |
| Calculator   | `numbat = "1.23"`          | Full unit-aware expression language        |
| Bundling     | `cargo-bundle`             | Produces `Guimbat.app` on macOS            |
| Icon         | `assets/make_icon.swift`   | Core Graphics; regenerate → see README     |

The `advanced` feature of iced **must** be enabled (see `Cargo.toml`) — it gates
`iced::advanced::text`, which is required for the custom syntax highlighter.

## File layout

```
src/main.rs          — entire application (single file, ~500 lines)
assets/
  make_icon.swift    — Swift script that draws the app icon from scratch
  icon_1024.png      — 1024×1024 source PNG (generated by the script)
  icon.icns          — macOS icon bundle (all sizes; referenced in Cargo.toml)
  AppIcon.iconset/   — intermediate per-size PNGs used by iconutil
Cargo.toml           — iced + numbat deps; cargo-bundle metadata
```

## Architecture (iced 0.13 Elm pattern)

```
init()          →  (Guimbat, Task<Message>)
update(state, msg) → Task<Message>
view(state)     →  Element<'_, Message>
subscription()  →  Subscription<Message>   (global arrow-key listener)
```

`Task` is iced's side-effect type; return `Task::none()` when there is nothing
async to do. Focus operations use `text_input::focus(text_input::Id::new(ID))`.

## State struct (`Guimbat`)

```rust
struct Guimbat {
    ctx: numbat::Context,       // stateful interpreter
    input: String,              // current text-input value
    history_log: String,        // accumulated output, newline-separated
    color_spans: Vec<ColorSpan>,// per-token byte-range colour annotations
    history_content: text_editor::Content, // iced model for the history pane
    highlight_settings: HighlightSettings, // passed to the syntax highlighter
    recall: Vec<String>,        // submitted expressions for ↑/↓ recall
    recall_cursor: Option<usize>,
}
```

## Syntax highlighting

The history pane uses a custom `GuimbatHighlighter` that implements
`iced::advanced::text::Highlighter`. Key points:

- `ColorSpan { start, end, color }` stores byte ranges over `history_log`.
- `HighlightSettings` carries the spans plus a `line_starts: Vec<usize>` (byte
  offset of each line's first char) so the highlighter can convert global offsets
  to line-local ones. `PartialEq` is implemented by comparing only the
  `generation: u64` counter — iced uses this to decide when to rebuild.
- `sync_history()` must be called after every change to `history_log`. It
  rebuilds `text_editor::Content`, moves the cursor to `DocumentEnd` (auto-scroll),
  recomputes `line_starts`, and increments `generation`.
- The correct method on `TextEditor` is **`highlight_with`**, not `highlight`.
  The format closure returns `text_core::highlighter::Format { color, font }`.

## History pane — key behaviour details

The history `text_editor` is **read-only** but selectable/copyable.

- Mouse events (`Click`, `Drag`, `Scroll`, `Move`, `Select`) are forwarded to
  `history_content.perform(action)` via `Message::HistoryAction`.
- Edit actions are swallowed (`action.is_edit()` guard).
- A custom `key_binding` closure handles keyboard focus logic:
  - **Not focused** → `None` (don't interfere).
  - **Cmd+C** → `Binding::Sequence([Copy, Custom(FocusInput)])` — copy then
    return keyboard focus to the text input.
  - **Printable character** → `Custom(TypeChar(s))` — appends the char to
    `state.input` and focuses the text input, so typing while the history pane
    is focused is seamlessly forwarded to the input field.
  - **Modifier-only keys** (Cmd, Shift, Alt, Ctrl, Meta) → `None` — must not
    redirect focus prematurely so that chords like Cmd+C still work.
  - **All other keys** → `Custom(FocusInput)`.

> **Gotcha:** `text_editor::Action` has no `Copy` variant. Copy is a
> `text_editor::Binding`, handled inside `key_binding`, not `on_action`.

## Arrow-key history recall

Implemented as a **global** `iced::keyboard::on_key_press` subscription, not
inside the text input widget. This means recall works regardless of which widget
currently has focus.

## Numbat integration

```rust
// Initialise (loads the standard prelude silently):
let mut ctx = Context::new(BuiltinModuleImporter {});
ctx.interpret_with_settings(&mut silent_settings, "use prelude", CodeSource::Internal);

// Evaluate user input:
ctx.interpret_with_settings(&mut settings, &raw, CodeSource::Text)
// → Ok((_stmts, InterpreterResult::Value(_) | InterpreterResult::Continue))
// → Err(e)  — display e with ERROR_COLOR
```

`print()` calls inside Numbat expressions are captured via the `print_fn`
callback in `InterpreterSettings` (collected into a `Vec<Markup>` behind an
`Arc<Mutex<_>>`).

## Common commands

```sh
cargo run                          # dev build + run
cargo build --release              # release build
cargo bundle --release             # build + create Guimbat.app
```

To regenerate the app icon after editing `assets/make_icon.swift`:

```sh
cd assets
swift make_icon.swift
mkdir -p AppIcon.iconset
for size in 16 32 64 128 256 512; do
  sips -z $size $size icon_1024.png --out AppIcon.iconset/icon_${size}x${size}.png
  sips -z $((size*2)) $((size*2)) icon_1024.png --out AppIcon.iconset/icon_${size}x${size}@2x.png
done
iconutil -c icns AppIcon.iconset -o icon.icns
```
